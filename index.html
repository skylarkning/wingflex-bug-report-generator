Read before you proceed

Complete the form after acknowledging these instructions, then copy the generated Markdown. Submit the report in one place only: either open the Wingflex Bridge Software Issues page , click New issue, and paste the Markdown into the issue description, or join the Wingflex Discord and paste the Markdown into the #bridge-bug-tracker channel.

Compatibility requests

The latest compatibility list can be found on wingflex.com under the item page for your specific product. If you would like to request support for an additional aircraft, add-on, simulator, or feature, choose File a Feature/Compatibility Request below.

Suspect a hardware issue? Do not fill out this form. Reach out to Wingflex Support on Discord for direct assistance instead.

Check for an existing report first

Before filing, search the existing issues in the wingflexsim/Bridge-Software repository for the same or a closely related problem.

Do not file the issue again if a duplicate or existing report is found. Duplicate reports slow down the team's workflow. Add any useful new details to the existing issue instead.

No GitHub or Discord account?

Complete the form and download the generated Markdown file. Email it to skyning@wingflexsim.com, use the report title you entered as the email subject, and attach the downloaded Markdown file. The file will be named bug_username_date.md for bugs or feature_request_username_date.md for requests.
